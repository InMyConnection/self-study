Командный процессор - специальная программа, которая запрашивает у пользователя команды и выполняет их.
Файловая система - способ организации информации на диске, при котором она разделяется на файлы, т. е. наборы данных, объединенные общим смыслом, записанные в одном или нескольких секторах.
Операционная система (operating system) - комплкс взаимосвязанных программ, предназначенных для управления ресурсами компьютера и организации взаимодействия с пользователем.
Каталог (directory) - область диска, где хранятся имена и другая важная информация о файлах. Каждому файлу соответствует элемент каталога (directory entry).
Чтобы запустить операционную систему, её нужно переписать из диска в память и запустить. Процесс называется загрузкой (booting).
Базовая система ввода-вывода (Basic Input/Output System, BIOS), базовая дисковая ОС (Basic Disk OS, BDOS), коммандный процессор консоли (Console Command Processor). Консоль - совокупность клавиатуры и дисплея.
Утилиты (utilities) - служебные небольшие программы, предназначенные для решения простых задач. Для решения более сложных задач служат прикладные программы (applications).
ОС обеспечивает прикладные программы интерфейсом прикладного программирования (Application Programming Interface, API). API является аппаратно независимым, обеспечивает программа легкий доступ к оборудованию компьютераю
Таблица размещения файлов (File Allocation Table, FAT) - файловая система, в которой дисковое пространство разделено на кластеры (clusters). Расширение файла (extension).
В иерархической файловой системе некоторые файлы сами могут быть каталогами. Обычный дисковый каталог называется корневым (root), а каталоги, вложенные друг в друга - подкаталогами (subdirectories).
ОС, способная одновременно выполнять много задач, называется многозадачной (multitasking).
Виртуальная память (virtual memory) - ненужные блоки памяти сохраняются на диске,  по мере надобности возвращаются обратно в память.
Проект GNU (GNU's not UNIX). Программы GNU совместимы с UNIX.
